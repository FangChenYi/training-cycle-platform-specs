## 4.1 功能規格 FSD

### FSD-01 教練建立 / 更新學員課表

| 項目       | 內容                                                                                                                                                                                                                                            |
| ---------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 目的       | 教練為學員建立或更新一份完整課表（含週、動作組、動作項），取代過去多人多份 Sheet 的維護方式                                                                                                                                                     |
| 主要角色   | Coach（建立 / 更新）；Athlete（事後只讀查看）                                                                                                                                                                                                   |
| 觸發       | 教練在學員頁或模板頁點「新增課表 / 編輯課表」                                                                                                                                                                                                   |
| 前置條件   | 已登入並具 Coach 角色；若選「複製學員過去課表」，該學員至少需有一份歷史課表                                                                                                                                                                     |
| 主流程     | 1. 教練選建立方式：套模板 / 複製學員過去課表 / 從空白建立<br/>2. 選課表模式（固定 / 週期）<br/>3. 來源課表載入（後兩種：挑來源並複製為新副本）或進入空白狀態<br/>4. 微調基本資料、日期區間、動作組、動作項<br/>5. 送出 → 系統一次性寫入整份課表 |
| 替代流程   | 編輯既有課表：跳過步驟 1-3，直接載入原課表後微調                                                                                                                                                                                                |
| 後置條件   | 系統存在一份完整課表；模板分支不綁定學員，學員副本綁定指定學員                                                                                                                                                                                  |
| 例外與錯誤 | 詳見 [4.3 介面契約 § 課表整批建立](#contract-create-full-cycle) 錯誤碼表                                                                                                                                                                        |
| 驗收準則   | (1) 三種建立方式都能產出可被排程引用的課表 <br/> (2) 模板修改不影響已建立的學員副本 <br/> (3) 整批寫入為單一交易，無半成品狀態 <br/> (4) 同學員同日無法被排到兩次同一週                                                                         |

### FSD-02 VALD Profile 同步

| 項目       | 內容                                                                                                                                                                                                                                                                                                       |
| ---------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 目的       | 從 VALD 拉入指定 Group 的所有 Profile，為尚未在本系統存在的 Profile 自動建立 Athlete 帳號                                                                                                                                                                                                                  |
| 主要角色   | Coach                                                                                                                                                                                                                                                                                                      |
| 觸發       | 教練在學員列表頁點「從 VALD 同步」（可選 Group 或全部）                                                                                                                                                                                                                                                    |
| 前置條件   | 已登入並具 Coach 角色；系統已配置可用的 VALD Client 認證                                                                                                                                                                                                                                                   |
| 主流程     | 1. 系統向 VALD 取得 Group 內 Profile 清單<br/>2. 比對本地：略過已建立帳號的 Profile<br/>3. 對新 Profile 逐筆向 VALD 取詳細資料（含 email、姓名、生日、性別、Group）<br/>4. 過濾出有 email 的 Profile<br/>5. 為每筆建立 Athlete 帳號（以「姓+名」為全名、預設密碼依規則組合）<br/>6. 回傳成功清單與失敗清單 |
| 替代流程   | Email 重複導致建立失敗 → 將該筆列入失敗清單並顯示「請至 VALD 修改 Email」，不中斷整體流程                                                                                                                                                                                                                  |
| 後置條件   | 成功筆：本系統多出對應的 Athlete 帳號，已綁定 VALD profile id<br/>失敗筆：未建立帳號，原 Profile 與失敗原因可顯示給教練                                                                                                                                                                                    |
| 例外與錯誤 | VALD 認證 / 拉取失敗整段中止；個別 Profile 缺 email 或重複 email 僅該筆失敗。詳見 [4.3 介面契約 § 同步 VALD Profile](#contract-sync-profiles)                                                                                                                                                              |
| 驗收準則   | (1) 同一個 Profile 不會被建第二次 <br/> (2) 失敗筆能明確指出原因（缺 email / email 重複 / 其他） <br/> (3) 本系統不會建立 / 修改 VALD 端的資料                                                                                                                                                             |

> 本流程不涵蓋從零建立 VALD Profile；採用單向設計的原因見 [02-analysis.md § VALD Profile 同步策略](02-analysis.md#vald-profile-同步策略從雙向到單向)。

### FSD-03 VALD 檢測同步

| 項目       | 內容                                                                                                                                                                                                                                                                                                                                                                                        |
| ---------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 目的       | 將指定學員在 VALD 上的最新檢測結果（SJ / CMJ / IMTP / HJ）增量拉回本系統並算出 DSI / EUR                                                                                                                                                                                                                                                                                                    |
| 主要角色   | Coach                                                                                                                                                                                                                                                                                                                                                                                       |
| 觸發       | 教練在學員的檢測頁點「同步」                                                                                                                                                                                                                                                                                                                                                                |
| 前置條件   | 已登入並具 Coach 角色；該學員已綁定 VALD profile id                                                                                                                                                                                                                                                                                                                                         |
| 主流程     | 1. 系統以本地此學員最新檢測時間為起點（沒有 → 用預設起點）<br/>2. 向 VALD 拉取此時間後新增 / 異動的 Tests<br/>3. 對每筆 Test：若測驗類型在支援白名單，再拉 trial 明細<br/>4. 過濾出白名單內的有效指標，計算各指標的平均與最大值<br/>5. 全部處理完後，算出 DSI、EUR<br/>6. 與本地比對：未變動 → 僅更新「上次同步時間」；新 / 有更動 → 寫入本地快照<br/>7. 回傳本次處理的測驗清單與 DSI / EUR |
| 替代流程   | 拉 trial 遇 VALD 限流（429）→ 退避後重試最多 3 次；仍失敗則整段中止並回報                                                                                                                                                                                                                                                                                                                   |
| 後置條件   | 本地檢測快照表反映 VALD 截至本次同步時間的狀態；後續查詢數據不必再打 VALD                                                                                                                                                                                                                                                                                                                   |
| 例外與錯誤 | 學員未綁定 profile id / VALD 拉取 / trial 拉取失敗對應不同錯誤碼。詳見 [4.3 介面契約 § 同步 VALD 檢測](#contract-sync-tests)                                                                                                                                                                                                                                                                |
| 驗收準則   | (1) 同一筆檢測重複同步不會在本地表產生重複列 <br/> (2) 白名單外的指標不會進入本地表 <br/> (3) DSI / EUR 計算結果與手算一致 <br/> (4) VALD 暫時不可用時，歷史查詢仍能走本地快照                                                                                                                                                                                                              |

## 4.3 介面契約

### 共通規則

| 主題       | 規則                                                                                                           |
| ---------- | -------------------------------------------------------------------------------------------------------------- |
| 傳輸       | HTTPS / JSON；UTF-8                                                                                            |
| 認證       | 除 `/auth/*` 與少數匿名端點外，所有 API 要求 `Authorization: Bearer <jwt>`                                     |
| 角色授權   | 端點明確標記可進入的角色                                                                                       |
| 回應結構   | 統一三段式：`Result`（成功旗標）、`Err`（失敗時的錯誤碼 + 訊息）、`Data`（成功時的資料）                       |
| 錯誤碼分類 | 1xxx 輸入 / 2xxx 認證 / 3xxx 授權 / 4xxx 資源不存在 / 5xxx 系統 / 6xxx 業務操作 / 7xxx VALD 整合 / 9xxx 未分類 |
| 時間格式   | 日期欄位 `YYYY-MM-DD`；時間戳 ISO-8601 UTC（`...Z`）                                                           |
| 分頁       | 列表型查詢一律提供 `Page` / `PageSize`；省略則代表「查全部」                                                   |
| 限流       | `/auth/*` 套用獨立的限流規則，防止暴力試密                                                                     |

### <a id="contract-create-full-cycle"></a>端點 1：課表整批建立 / 更新

**路由**

- 建立：`POST /training/cycle/create-full`
- 更新：`POST /training/cycle/update-full/{cycleId}`

**授權**：Admin, Coach

**請求 Body 主要欄位**

| 欄位                                                                                                     | 必填   | 型別          | 說明                                                                  |
| -------------------------------------------------------------------------------------------------------- | ------ | ------------- | --------------------------------------------------------------------- |
| `userId`                                                                                                 | 否     | int?          | 空 = 模板；有值 = 學員專屬副本                                        |
| `sectionType`                                                                                            | 是     | int           | 課表分類（暖身 / 活動度 / 核心 / 體能 / 肌力 / 小肌群 / 技術 / 其他） |
| `name`                                                                                                   | 是     | string (≤100) | 課表名稱                                                              |
| `startDate`                                                                                              | 是     | date          | 起日                                                                  |
| `endDate`                                                                                                | 是     | date          | 迄日，須 ≥ `startDate`                                                |
| `sourceCycleId`                                                                                          | 否     | int?          | 來源課表 ID（套模板 / 複製過去課表時帶；空白建立則不帶）              |
| `cycleWeekList`                                                                                          | 是     | array (≥ 1)   | 週清單                                                                |
| `cycleWeekList[].weekNumber`                                                                             | 是     | int           | 第幾週；同一份課表內不可重複；`0` 表示固定課表                        |
| `cycleWeekList[].name`                                                                                   | 否     | string        | 週名稱，省略時系統 fallback 為「Week N」                              |
| `cycleWeekList[].groups`                                                                                 | 是     | array         | 動作組清單                                                            |
| `groups[].name` / `rounds` / `restBetweenRoundsSeconds` / `order`                                        | 視欄位 | 各型別        | 組名稱、回合數、組間休息秒數、組順序                                  |
| `groups[].exerciseItemList`                                                                              | 是     | array         | 該組底下的動作項                                                      |
| `exerciseItemList[].exerciseId` / `name` / `sets` / `reps` / `seconds` / `intensity` / `notes` / `order` | 視欄位 | 各型別        | 動作項細節                                                            |

**回應**：成功時回完整 `CycleDetail`（含展開的 weeks → groups → items）

**錯誤回應**

| 錯誤碼 | 情境                                            |
| ------ | ----------------------------------------------- |
| 1102   | `startDate > endDate`                           |
| 1100   | 週清單為空                                      |
| 1101   | 週號在同一份課表內重複                          |
| 1301   | 模板分支下，同分類存在同名課表                  |
| 4000   | 帶了 `userId` 但學員不存在 / 更新時找不到該課表 |
| 6000   | 寫入失敗（DB 層）                               |

### <a id="contract-sync-profiles"></a>端點 2：同步 VALD Profile

**路由**：`POST /athlete/vald/{groupId?}/sync-profiles`

**授權**：Admin, Coach

**請求**：路徑參數 `groupId`（可省略代表全部 Group），無 Body

**回應 Body 主要欄位**

| 欄位                       | 型別             | 說明                                                        |
| -------------------------- | ---------------- | ----------------------------------------------------------- |
| `successProfiles`          | array of athlete | 本次成功建立的學員，包含本地 ID、Email、姓名、Group、性別等 |
| `failedProfiles`           | array of failure | 本次未能建立的 Profile                                      |
| `failedProfiles[].profile` | object           | 該筆 VALD Profile 的原始資料                                |
| `failedProfiles[].error`   | string           | 失敗原因（如 "Email 重複，請至 VALD 修改"）                 |

**錯誤回應**

| 錯誤碼            | 情境                                           |
| ----------------- | ---------------------------------------------- |
| 7102              | 向 VALD 取 Profile 列表 / 詳細失敗（整段中止） |
| 7001 / 7002       | VALD 認證 / Token 失效                         |
| 個別 Profile 失敗 | 不回錯誤碼，列入 `failedProfiles` 並繼續       |

### <a id="contract-sync-tests"></a>端點 3：增量同步 VALD 檢測

**路由**：`GET /athlete/vald/{athleteId}/sync-tests`

**授權**：Admin, Coach

**請求**：路徑參數 `athleteId`，無 Body 與 Query

**回應 Body 主要欄位**

| 欄位                                          | 型別               | 說明                                            |
| --------------------------------------------- | ------------------ | ----------------------------------------------- |
| `dsi`                                         | number? (2 位小數) | 本次處理測驗範圍內計算出的 DSI；無法計算則 null |
| `eur`                                         | number? (2 位小數) | EUR 同上                                        |
| `tests`                                       | array              | 本次處理的測驗清單                              |
| `tests[].testId`                              | string             | 測驗 ID                                         |
| `tests[].testType`                            | string             | `SJ` / `CMJ` / `IMTP` / `HJ`                    |
| `tests[].recordedDateUtc` / `modifiedDateUtc` | ISO-8601 UTC       | 檢測時間 / 異動時間                             |
| `tests[].weight`                              | decimal?           | 體重（可缺）                                    |
| `tests[].averages`                            | object             | 白名單指標 → 平均值（2 位小數）                 |
| `tests[].maxes`                               | object             | 白名單指標 → 最大值（2 位小數）                 |
| `tests[].trials`                              | array              | 過濾後的 trial 明細                             |

**支援的測驗類型與白名單指標**

| 測驗類型 | 白名單指標                                                                                                                                                                 |
| -------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| SJ       | `JUMP_HEIGHT_IMP_MOM`、`PEAK_TAKEOFF_POWER`、`PEAK_TAKEOFF_FORCE`、`CONCENTRIC_IMPULSE_P1` / `P2`、`CONCENTRIC_RFD_200`、`RSI_MODIFIED_IMP_MOM`                            |
| CMJ      | `JUMP_HEIGHT_IMP_MOM`、`ECCENTRIC_DECEL_IMPULSE`、`ECCENTRIC_PEAK_VELOCITY`、`CONCENTRIC_IMPULSE_P1` / `P2`、`PEAK_CONCENTRIC_FORCE`、`PEAK_TAKEOFF_FORCE`、`RSI_MODIFIED` |
| IMTP     | `BW_NET_FORCE_PEAK`、`PEAK_VERTICAL_FORCE`、`RFD_AT_250MS`、`START_TO_PEAK_FORCE`、`START_TO_80PC_PEAK_FORCE`                                                              |
| HJ       | `HOP_RSI`                                                                                                                                                                  |

**衍生指標計算**

| 指標 | 公式                                                                                                 |
| ---- | ---------------------------------------------------------------------------------------------------- |
| DSI  | CMJ 平均 `PEAK_TAKEOFF_FORCE` ÷ IMTP 平均 `PEAK_VERTICAL_FORCE`（缺則 fallback `BW_NET_FORCE_PEAK`） |
| EUR  | CMJ 平均 `JUMP_HEIGHT_IMP_MOM` ÷ SJ 平均 `JUMP_HEIGHT_IMP_MOM`                                       |

**錯誤回應**

| 錯誤碼 | 情境                                              |
| ------ | ------------------------------------------------- |
| 4000   | 學員不存在 / 未綁定 VALD profile id               |
| 7200   | 向 VALD 取 Tests 失敗                             |
| 7201   | 拉 trial 明細失敗（含 429 重試 3 次後仍失敗）     |
| 7302   | VALD 限流（內部會自動退避重試，最終仍失敗才外露） |
